# PRD — CapCut 유사 데스크톱 영상 편집기

> 제품 요구사항 정의서 (Product Requirements Document)
> 최종 수정: 2026-07-02 · 상태: 기획

## 1. 제품 개요

숏폼~중편 영상을 쉽게 편집·내보내기 할 수 있는 데스크톱 영상 편집기.
CapCut의 낮은 진입장벽을 지향하되, 데스크톱의 성능과 정밀 편집을 결합한다.

- **플랫폼**: Windows / macOS 데스크톱 (Electron)
- **목표 사용자**: 숏폼 크리에이터, 1인 미디어, 소규모 마케팅 팀
- **제품 성격**: 실제 배포/수익화를 목표로 하는 상용 제품

## 2. 문제 정의 & 목표

| 문제 | 이 제품의 접근 |
|------|----------------|
| 전문 편집기(Premiere 등)는 진입장벽이 높다 | 직관적 타임라인 + 프리셋 중심 UX |
| 모바일 편집기는 정밀 편집·성능에 한계 | 데스크톱 성능 + 프레임 단위 편집 |
| 미리보기와 결과물이 다르다 | WYSIWYG 렌더링 원칙 (핵심 차별점) |

### 성공 지표 (초기, 수치 기준)
- 임포트 → 컷 편집 → 자막 → 내보내기까지 **끊김 없는 1회 완주** 가능 (e2e 스모크 자동화, dev-plan 1.6)
- **내보내기 속도**: 1080p30 5분 타임라인 ≤ **2× 실시간(10분)** — 기준 개발 머신에서 측정
- **WYSIWYG**: 기준 프레임 **SSIM ≥ 0.99**, 출력 duration 오차 ±1프레임, A/V 싱크 ±40ms
  (정성적 "비슷해 보임"이 아니라 자동 비교 스크립트로 판정 — dev-plan 5.2.5)

## 3. 범위 (MVP)

### In Scope (핵심 4대 기능)
1. **타임라인 컷 편집** — 임포트, 자르기/분할, 이어붙이기, 트림, 멀티트랙, 스냅
2. **자막/텍스트** — 텍스트 오버레이, 스타일, 등장/퇴장 애니메이션, (선택) 자동 자막
3. **오디오/배경음악** — 오디오 트랙, 볼륨/페이드, 다중 트랙 믹싱, 파형
4. **효과/전환/필터** — 기본 색보정 필터, 클립 간 전환, GPU 실시간 프리뷰

### Out of Scope (초기 제외)
- 멀티캠 편집, 키프레임 애니메이션 전면 지원
- 협업/클라우드 동기화
- 모바일 앱
- AI 생성(영상 생성, 아바타 등) — 자동 자막(STT)만 선택적 포함

## 4. 사용자 시나리오 (핵심 플로우)

1. 사용자가 영상 파일 여러 개를 드래그앤드롭으로 임포트
2. 타임라인에 배치하고 불필요한 구간을 잘라냄
3. 배경음악을 깔고 원본 소리 볼륨을 조절
4. 자막을 추가 (또는 자동 자막 생성 후 편집)
5. 색 필터와 클립 전환 효과 적용
6. 미리보기로 확인 → 해상도/포맷 선택 → 내보내기

## 5. 기능 요구사항

### 5.1 타임라인 편집
- 여러 포맷(H.264/H.265/최소한의 코덱) 임포트
  - H.265 등 WebCodecs 미지원 환경(예: Windows HEVC)과 가변 프레임레이트(VFR) 소스는 호환 프록시(H.264 CFR)로 자동 변환 후 편집 (dev-plan 0.4)
- 프레임 단위 정확 편집 (자르기/분할/트림)
- 멀티 비디오/오디오/텍스트 트랙
- 클립 경계·플레이헤드 스냅
- undo/redo (전체 편집 조작)

### 5.2 재생/프리뷰
- 실시간 재생, 일시정지, 스크럽
- 프레임 정확 시크 (±1 프레임)
- A/V 동기화 (±40ms 이내)

### 5.3 자막/텍스트
- 폰트/크기/색/정렬/외곽선/그림자
- 표시 구간 지정, 화면 내 위치·크기 조절
- 등장/퇴장 애니메이션
- (선택) 음성 인식 기반 자동 자막 + 편집

### 5.4 오디오
- 오디오 클립 배치/트림, 파형 표시
- 클립/마스터 볼륨, 페이드 인/아웃
- 다중 트랙 믹싱 (클리핑 방지)

### 5.5 효과/전환/필터
- 밝기/대비/채도/색온도 등 기본 필터
- 디졸브/와이프/슬라이드 전환
- 실시간 GPU 프리뷰, 파라미터 저장

### 5.6 내보내기
- 해상도/비트레이트 프리셋 — **초기 포맷은 H.264 + AAC (MP4) 한정** (H.265 인코딩은 특허 라이선스 검토 후 별도 결정)
- 진행률 표시 + 취소
- **프리뷰와 결과물 일치 (WYSIWYG)** — SSIM 기준 자동 검증

### 5.7 프로젝트 관리
- 프로젝트 저장/열기, 자동저장, 크래시 복구
- 미디어 상대경로 관리, 누락 파일 재연결

## 6. 비기능 요구사항
- **성능**: 고해상도 소스는 프록시(저해상도) 편집 지원
- **안정성**: 장시간 세션 메모리 안정, 크래시 복구
- **품질 기준**: 프레임 정확도, A/V 싱크, WYSIWYG (상세는 dev-plan.json 검증 기준 참조)
- **배포**: 코드사이닝/공증, 자동 업데이트

## 7. 리스크 (기술)
1. 프레임 정확 시크 (WebCodecs 키프레임 처리)
2. A/V 동기화
3. 프리뷰 ≠ 결과물 (WYSIWYG) — **완화: 내보내기 스파이크를 Phase 1.5로 전진 배치해 구조 리스크 조기 검증**
4. 내보내기 프레임 전송 처리량 (1080p30 RGBA ≈ 초당 250MB, IPC/파이프 병목) — 1.5 스파이크에서 실측
5. 코덱 호환성 (Windows HEVC 등 WebCodecs 편차, VFR 소스) — 완화: 호환 프록시 (Phase 0.4)
6. 고해상도 성능 (성능용 프록시 필수, Phase 6.2)
7. FFmpeg 필터그래프/렌더 명세 변환 정확도

자세한 기술 설계는 [ARCHITECTURE.md](./ARCHITECTURE.md), 단계별 실행 계획은 [`../dev-plan.json`](../dev-plan.json) 참조.
