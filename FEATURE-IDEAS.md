# 향후 기능 아이디어 (검토용 메모)

phase-8(다중 선택 일괄 편집)·phase-9(전환 효과음) 이후 추가 개발 후보. 2026-07-10 정리.
앱 성격(자막·전환 효과음·무음 자동컷·다중 선택)상 **숏폼/소셜 영상 제작**이 주 타깃이라는 전제로 우선순위를 매김.

> 이 중 **4. 속도 조절**과 **5. 자막 SRT 가져오기**는 개발 완료 (2026-07-10). 나머지 1·2·3 및 배포 블로커는 미착수.

## 1. 키프레임 애니메이션 ⭐ (최우선 후보)
- **무엇**: 클립의 `transform`(크기/위치/회전)·`opacity`·필터 값을 시간에 따라 변화. 켄번즈(줌인/팬), 움직이는 자막, PIP 이동.
- **왜 지금**: 지금은 `transform` 이 클립당 고정값. 컴포지터가 이미 **시각별 샘플링 렌더**(전환도 동일 방식)라 보간만 얹으면 프리뷰·내보내기 양쪽에 자동 반영.
- **규모/난이도**: 중상. 데이터 모델(키프레임 트랙)·보간 함수·타임라인 키프레임 UI 필요.
- **가치**: 최상. "장난감 → 진짜 편집기" 분기점.

## 2. 세로/정사각 종횡비 + 내보내기 프리셋 확대
- **무엇**: 9:16(쇼츠/릴스)·1:1·GIF 프리셋. 프로젝트 종횡비 설정 + 내보내기 프리셋.
- **왜 지금**: 프리뷰 16:9 고정(styles.css), 내보내기는 해상도 축소만. 내보내기 스케일 로직이 이미 있어 상대적으로 저비용.
- **규모/난이도**: 중. 프로젝트 settings 에 종횡비 + 프리뷰 캔버스/세이프영역 대응.
- **가치**: 상. 타깃 사용자에게 직결.

## 3. BGM 오토 더킹 (말할 때 배경음 자동 감소)
- **무엇**: 음성 구간에 배경음악 트랙 볼륨을 자동으로 낮춤(사이드체인 유사).
- **왜 지금**: 기존 무음 감지(FFmpeg silencedetect) + Web Audio 게인 그래프 재활용.
- **규모/난이도**: 중. 음성 구간 검출 → 배경음 트랙 게인 자동화(키프레임형).
- **가치**: 상. 체감 품질 크게 상승.

## 4. 속도 조절 / 슬로모 — ✅ 완료 (2026-07-10)
- **구현**: `setClipSpeed` 커맨드 — 불변식 3 유지 위해 소스 구간 고정·타임라인 길이 재계산, 같은 트랙 뒤 클립 리플(상대 간격 보존), 0.25~4배 클램프, 전환 길이 초과 시 클램프. 인스펙터에 배속 입력 + 0.5×/1×/2× 프리셋(단일 선택 · 비디오/오디오). 유닛 6 + e2e.
- **남은 것(후속)**: 구간별 속도 램프는 키프레임(1번) 선행 필요 — 미구현.

## 5. 자막 SRT 가져오기 — ✅ 완료 (2026-07-10)
- **구현**: `parseSrt`(shared, 순수 — CRLF·다중행·번호누락·`,`/`.` ms 견고, 잘못된 블록 스킵, 정렬) + `subtitles:openSrt` IPC(파일 열기) + `importSubtitlesFromSrt`(겹침 클램프 + createSubtitleClip + addSubtitleClips 단일 undo) + TransportBar "⬇ 자막" 버튼. 유닛 6(왕복 포함) + e2e.
